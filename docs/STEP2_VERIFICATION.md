## C. 重複登録防止（L1/L2/L3）

### L1: 同日・同メール・確認済 → ブロック
- [ ] **C1**: ユーザーA で日付X に予約作成 → 成功
- [ ] **C2**: 同じユーザーで同じ日付X にもう一度予約 → `OK ボタンでマイ予約、キャンセルでとどまる` の confirm ダイアログが表示
- [ ] **C3**: メッセージに既存予約番号が含まれている
- [ ] **C4**: OK で「マイ予約」画面に遷移する

### L2: 5分以内の連投 → ブロック
- [ ] **C5**: 別日Y にユーザーA で予約 → 成功
- [ ] **C6**: 直後（5分以内）に同じユーザーで違う日付Z に予約 → これは別日なので L2 にならず成功（L2 は同日のみ）
- [ ] **C7**: ユーザーA の Y 日付予約をキャンセル → 利用者マスタ更新確認
- [ ] **C8**: キャンセル直後（5分以内）に同じユーザーで Y 日付に再予約 → L2 ブロック「⚠️ 重複送信を検出しました」トースト表示

### L3: 利用者マスタ blocked → ブロック
- [ ] **C9**: 利用者マスタシートで該当ユーザー行のK列「ブロックフラグ」を `TRUE` に変更
- [ ] **C10**: そのユーザーで新規予約しようとする → L3 ブロック「🚫 お電話でお問合せください」エラー表示
- [ ] **C11**: K列を `FALSE` に戻す → 再び予約可能
