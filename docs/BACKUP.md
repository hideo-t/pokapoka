# データバックアップ運用手順

> Step 2 以降で `updateUserMaster()` 統合・重複防止追加など、データ構造に再度触る作業が発生する。
> **本番データがある状態での migration は、必ずバックアップを取ってから** が運用ルール。

---

## 1. 手動バックアップ（推奨・週1回）

### 手順
1. 対象スプレッドシートを開く: <https://docs.google.com/spreadsheets/d/15Yw5uQejs4Y0XkKu8t_ILsB7nP-8x1ZeIda4tKGugjw/edit>
2. 上部メニュー **ファイル → コピーを作成**
3. ファイル名を `ぽかぽか食堂_予約データ_backup_YYYY-MM-DD` に変更
4. 保存先 Drive フォルダを指定（推奨: `予約システム/バックアップ/` フォルダを事前に作る）
5. 「コピーを作成」

### バックアップが必要なタイミング
| タイミング | 理由 |
|---|---|
| 毎週月曜（または週末）| 通常運用での定期バックアップ |
| `initializeSheets()` を実行する直前 | 列追加・削除 migration で事故った場合の復旧用 |
| `clasp push` で大きな変更を反映する直前 | コードバグでデータ破壊する可能性に備える |
| 新規機能リリース直前（Step 2 / Step 3 着手前）| 動作確認後に問題が出た場合の戻し用 |

## 2. Google ドライブの版の履歴（自動・90日保持）

### 確認方法
1. スプレッドシートを開く
2. **ファイル → 版の履歴 → 版の履歴を表示**
3. 右側に日時別の版がリストアップされる
4. 戻したい版をクリック → **「この版を復元」** で元に戻せる

### 注意点
- Google が自動で記録する版は **約90日間** 保持される
- 大量の編集で版が増えすぎると古い版は自動的に削除される
- 重要なチェックポイントは「**名前付きの版**」として保存推奨（メニューから「版に名前をつける」）

## 3. 自動バックアップ（Phase 2 で実装予定）

中央 GAS から月次トリガーで自動コピーを生成する機能を Phase 2 で追加。
```javascript
// 実装イメージ（Phase 2 で gas/Admin.gs or 中央GASに追加）
function backupSpreadsheetMonthly() {
  var src = SpreadsheetApp.openById(SPREADSHEET_ID);
  var date = Utilities.formatDate(new Date(), 'Asia/Tokyo', 'yyyy-MM-dd');
  var copy = src.copy(ORG_NAME + '_予約データ_backup_' + date);
  var folder = DriveApp.getFolderById(BACKUP_FOLDER_ID);
  DriveApp.getFileById(copy.getId()).moveTo(folder);
  // 3ヶ月以上前のバックアップは自動削除（任意）
}
```
それまでは **手動バックアップ運用** で対応。

## 4. 復旧手順

### ケースA: 直前の `initializeSheets` で列構造が壊れた
1. 版の履歴 → 実行前の版を選択 → 「この版を復元」
2. 復元後にコードのバグを修正 → 再度バックアップ → 再実行

### ケースB: `clearAllData` を誤って実行した
1. 版の履歴 → 実行直前の版を選択 → 「この版を復元」
2. それでも戻らない場合 → 手動バックアップファイルを開く → 各シートのデータをコピー → 本番に貼付

### ケースC: 個別の予約を誤って削除した
1. 版の履歴で対象行のあった版を確認
2. その版を開いて当該行を手動コピー
3. 本番に追記（予約IDが既存と被らないよう注意）
